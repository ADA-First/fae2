    Accessibility Evaluation of WCAG 2.0

    • The default rulesets used in the Functional Accessibility Evaluator 2.0 (FAE) are based the requirements of the Level A and Level AA success criteria of the Web Content Accessibility Guidelines (WCAG) 2.0.
    • The approach of FAE 2.0 is to support evaluation of not only accessibility, but also the usability of web resources by people with disabilities through the use of accessible design technques and the use of the latest accessibility techniques like the W3C Accessible Rich Internet Accessibility Specification.
    • There are many potential techniques to meet the requirements of WCAG 2.0 and the W3C WCAG 2.0 Techniques document provides examples of some of the techniques that can be used satisfy the requirements.
    • Not all of the accessibility techniques tested by FAE 2.0 are included in the W3C WCAG 2.0 Techniques document, but the techniques have been submitted to the W3C WCAG techniques working group for consideration for inclusion.

    Limitations

    Please be aware of the following limitations in the use of FAE:

    • While passing all of the FAE rules will generally provide highly accessible web pages, it should not be inferred from this that all accessibility requirements have been met. We encourage you to become familiar with all accessibility requirements of WCAG 2.0 and to be sure to implement them in your designs, including requirements that can only be tested manually.
    • FAE and its test components may not be able to interpret all markup; we recommend that you validate your documents to improve the ability of FAE to intepret markup.
    • If you are using accessibility techniques for conforming to WCAG 2.0 that are not identified in a ruleset, this may result in not passing some of the rules used for evaluating accessibility.
